# Eval-критерии для /infostyle

Механические, проверяемые да/нет вопросы. Без субъективных оценок.

---

## Критерии (7 вопросов)

### 1. Стоп-слова убраны?
В секции «Отредактированный текст» НЕТ слов из списка: очень, максимально, уникальный, качественный, инновационный, передовой, высококвалифицированный, данный, является, осуществлять, индивидуальный подход, команда профессионалов, широкий ассортимент. Проверка: grep по списку в секции «Отредактированный текст». В секции «Что изменилось» допускается описание стоп-слов своими словами (но не дословные цитаты).

### 2. Пустоты заполнены?
Результат НЕ короче оригинала более чем на 40%. Если оригинал 100 слов → результат минимум 60 слов. Исключение: UI кнопки, toggle, пуш-уведомления, системные и in-app уведомления, SMS, тултипы, пустые состояния, loading states (там краткость — цель).

### 3. Есть конкретика?
Результат содержит хотя бы одну из форм конкретики: число/цифра, единица измерения (минут, рублей, штук), конкретное название, конкретное действие. Проверка: наличие цифр или единиц.

### 4. Контекст определён?
Вывод содержит блок «Контекст» (или эквивалент) с указанием типа текста и аудитории. Проверка: grep по «Тип:» или «Аудитория:».

### 5. Формат вывода соблюдён?
Ответ содержит ВСЕ обязательные блоки: оценка до (4 числа), отредактированный текст, список изменений (минимум 1 пункт), оценка после (4 числа). Проверка: grep по «Ясность:» (минимум 2 вхождения — до и после), «Что изменилось», и «Оценка (после)».

### 6. Смысл сохранён?
Ключевые сущности оригинала присутствуют в результате. Если оригинал про «презентации» — результат тоже про презентации. Если про «удаление» — результат тоже про удаление. Проверка: ключевые существительные оригинала есть в результате.

### 7. Юридический текст не тронут?
Для юридических кейсов (33, 34, 35): скилл НЕ применил полную редактуру. Либо отказался, либо предупредил, либо применил минимальные изменения. Проверка: текст содержит предупреждение о юристе/точности/правовых рисках.

## Подсчёт

- 56 кейсов × 7 критериев = 392 максимум
- Критерий 2 автопрохождение для краткостных типов (кнопки, пуш, SMS, тултипы, loading)
- Критерий 7 применяется только к юридическим кейсам (33-35), для остальных = автопрохождение
- Score = сумма / total × 100 = процент
- Цель: 90%+
